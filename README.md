### ***리눅스 명령어 top, ps, jobs, kill에 대하여*** 

# **top**

리눅스 서버의 OS 상태를 확인할 수 있게 해주는 명령어

## *기능*

서버의 CPU 사용량 확인

메모리 사용량 확인

실행중인 프로세스를 모니터링

## *사용옵션*
| 옵션    | 설명       | 
|:----------:|:-----------:|
|  -d [초(숫자)] | 초 단위 갱신주기 설정  |
| -b| 배치 모드|
|-n [횟수] | 출력 횟수 지정|

## *단축키*
| 단축키    | 설명       | 
|:----------:|:-----------:|
| 1 | CPU 코어별 사용 현황     | 
|t | CPU 통계 변경 or 제거|
|m| 메모리 통계 변경 or 제거|
|h|도움말 확인|
|P|CPU 사용률 큰 순서로 정렬|
|M|메모리 사용량이 큰 순서로 정렬|
|k| 입력 후 종료할 PID 입력|
|c|명령어와 인자 표시 or 감추기|
|z| 출력 색상 변경|
|q|top 종료|

# **ps**

현재 실행 중인 프로세스와 상태를 출력

## *기능*

USER UID 출력

PID 출력

PPID 출력

STAT 출력

TTY 출력

VSZ 출력

RSS 출력

USER 출력

TIME 출력

COMMAND 출력

%CPU 출력

%MEM 출력

C 출력

STIME 출력
## *사용옵션*
| 옵션     | 설명       | 
|:----------:|:-----------:|
| -A |모든 프로세스를 출력 |
| a|모든 사용자의 프로세스 정보 표시 |
| -f|프로세스에 대한 정보 출력 |
|u|사용자 중심 프로세스 정보 출력|
|-u <사용자>|특정 사용자의 프로세스 정보를 출력 (사용자를 지정하지 않으면 현재 사용자를 기준으로)|
|x|터미널에 종속되지 않은 프로세스를 출력|
|-m|프로세스뿐만 아니라 커널 스레드도 출력|
|-r|현재 실행 중인 프로세스 출력|
# **jobs**

현재 쉘 세션에서 실행 중이거나 일시 중지된 프로세스를 확일할 때 사용

## *기능*

작업 번호 출력

PID 출력

작업 상태(Running, Stopped 등) 출력

명령어 출력

## *사용옵션*
| 옵션     | 설명       | 
|:----------:|:-----------:|
| -l|각 작업의 PID를 함께 출력 |
|-p |각 작업의 PID만 출력 |
| -n|가장 최근 상태가 변경된 작업만 출력 |
|-r |실행 중(Running)인 작업만 출력 |
|-s |일시 중지된 작업만 출 |

# **kill**

리눅스에서 프로세스에 시그널을 보내 종료하는 명령어

## *기능*

SIGTERM (15번) : 정상 종료 시그널(기본값)

SIGKILL (9번) : 강제 종료 시그널

SIGSTOP (19번) : 프로세스를 일시 정지

SIGCOUNT (18번) : 일시 정지된 프로세스를 재개

여기서 번호와 SIG를 제외한 단어인 시그널을 옵션으로 사용

PID는 top와 ps를 활용해 확인 가능

kill 명령어는 ps와 조합하는 경우도 있다

## *사용옵션*
| 옵션     | 설명       | 
|:----------:|:-----------:|
| -15 또는 -TERM [PID]|프로세스 정상 종료 |
| -9 또는 -KILL [PID] |프로세스 강제 종료  |
| -19 또는 -STOP [PID]| 프로세스 일시정지|
| -18 또는 -COUNT [PID]|일시정지된 프로세스를 재개 |
| -s [시그널 번호 또는 이름]| 보낼 시그널 지정 |
| -l|사용 가능한 모든 시그널 숫자와 이름을 보여줌 |
